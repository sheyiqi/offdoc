#### Mbist

[Company Name] memory compilers offer integrated Mbist functionality.

Within SRAM macro cells, the built-in MBIST (Memory Built-In Self Test) functions as a dedicated engine for memory array testing. Typically, third-party Mbist tools are employed to conduct this testing on SRAM instances. Its primary capability lies in autonomously performing full-speed, comprehensive fault screening of the SRAM's memory cell array, sense amplifiers, write drivers, and portions of the read path, without the need for external automatic test equipment (ATE) to supply high-precision memory test vectors. 

The Mbist functionality provided by [Company Name] memory compilers includes: 

- Incorporating an additional register chain into the Scan Chain: an output data flip-flop chain. 
- Augmenting the address/read-write enable/chip select enable trigger chain within the scan chain with triggers for signals such as PIPE, CDI[3:0], CAPTE, and STICE. 
- An Mbist comparator: It supplies expected output values for comparison with the data output from the SRAM Instance, executes the comparison, and outputs the results through an output data flip-flop. 

The built-in Mbist functionality of [Company Name]'s memory compilers encompasses a portion of the functionalities and hardware circuitry found in third-party Mbist solutions. It replaces a relatively small segment of the SRAM instance's circuit overhead with a more streamlined and less complex hardware circuitry alternative.

Upon initiating the Mbist test, the controller automatically implements the preset March algorithm (e.g., March C-, March LR, etc.) to systematically traverse all addresses at the operational frequency. It performs sequential ascending/descending operations such as "write 0→read 0→write 1→read 1" on each unit, concurrently comparing the read data with the expected values in real-time. Upon detecting a mismatch, the SRAM's built-in comparator promptly latches the faulty address and erroneous data, transmitting diagnostic information to the top-level test access port via an error flag signal or serial scan output. 

In contrast to scan chains, which primarily focus on enhancing the testability of peripheral logic, built-in MBIST directly tackles the testing challenges inherent in the memory array itself. It facilitates at-speed testing and can uncover faults that manifest solely under dynamic conditions (e.g., read disturb, write recovery failure, data retention failure), which are challenging to detect using low-speed external testing equipment. Moreover, MBIST often collaborates with built-in redundancy analysis (BIRA) and repair (BISR) mechanisms to ascertain in real-time during testing whether a fault can be rectified through redundant rows/columns. If feasible, it automatically programs fuses or configures repair registers, thereby directly enhancing the yield of finished products during the manufacturing phase. 

For external systems, the built-in MBIST encapsulates all intricate test logic within the SRAM macro, presenting only minimal interfaces such as clock, enable, and pass/fail indicators. This enables swift completion of a comprehensive memory function check during wafer testing, post-packaging final testing, and even system-level power-on self-test, substantially reducing testing costs and ATE dependencies. It represents a standard design approach for achieving high-reliability, high-yield SRAM in advanced SoCs. 

> Note: The Mbist feature is only available when the user selects Bist Mux and Scan Chain.
