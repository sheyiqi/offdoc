### LEF Model Usage

The `Layout Exchange Format (LEF)` provides an abstract representation for memories. This is the minimum information required by Cadence place and route tools to process the memory macro generated by the compiler. This abstraction defines the macro's dimensions, pin names and locations, pin layers, and (optional) diode/diffusion area information. It also includes all physical segments generated by the `Power and Pin Router (PPR)`, which can be associated with power routing or signal routing. You can also define obstruction regions in the LEF (areas that the router must avoid during global or detail routing). The macro core is marked as an obstruction to avoid capacitance coupling crosstalk with critical internal signals such as bit, bitbar, se, and seb.

The `LEF` view only contains `MACRO` definition. You must provide the technology definitions (layer names, layer properties and vias), and load them into the router before reading the `LEF` view.
